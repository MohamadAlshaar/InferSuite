## Corrections baked into this snapshot (vs the original runs)
1. **FP/FLOPs** — precision-specific lanes (128b = 4 SP / 2 DP, 256b = 8/4, 512b = 16/8) + **×2 FMA**.
   (Old bug: summed SP+DP then applied the DP lane count to both, and omitted FMA → SP-packed halved, MFLOP ~2× low.)
2. **DRAM** — **cgroup-scoped `L3-miss×64`** (container-only), NOT node-wide IMC.
   (Node-wide IMC was 3–26× inflated by OS/page-cache/background — these are L1-resident workloads.)
3. **TMA Level-2** — clean fixed-counter `td2` events (`topdown-fetch-lat/heavy-ops/mem-bound/br-mispredict`
   + derived siblings), replacing the degenerate `toplev` pass (which pruned to 3–5 nodes / 12 s windows).
4. All derivations go through **`microarch.py`** (one source of truth, can't drift).

## Caveats (read before citing)
- **BCB** is measured **system-wide** (`-a`, no container) with the box idle; SWE + OpenClaw are **cgroup-scoped** (`-G`) → cleaner.
- **arxiv_digest** is a bimodal web task (agent loops range 1–86). Its FP/TD2 come from medium-intensity runs
  while TMA/CACHE/MLP are heavy runs → per-group IPC is **not coherent**. Treat as best-effort. calendar is the clean one.
- **Node-wide IMC DRAM is NOT used here** (superseded). The old inflated values (SWE 13–22 GB, BCB 37 GB, arxiv 23 GB) are wrong.
- `toplev` passes are superseded by `td2` — ignore any `group_toplev.txt`.
